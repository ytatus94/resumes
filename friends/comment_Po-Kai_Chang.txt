我個人的建議
1. 用 PDF
   我這顯示 Intended grad: May. 2020 被斷成兩行
   不曉得是你原本就分兩行寫，還是我電腦顯示的問題
   所以最好用 PDF 才能確保招聘主管看到的格式與你電腦上顯示的相同
2. 有 Github 的連結也請放上去
3. Skills 分成幾類
   PL/SQL, SQL ---> 重複兩個 SQL
   Oracle 和 Linux 不適合放在 language 類別
   建議把 Orcale, SQL 放到一個叫做 database 的類別
   想找 data science 相關工作的話，其實沒必要放 HTML 和 XML
4. Skill 後面先寫 experience
   你的經驗都和 Data science 無關，不論你想找實習或是正職，
   工作經驗請放跟想找工作相關的，如果沒有其他相關經驗，那就只放專案就好
5. 專案的部分，既然你有 kaggle 的專案，建議可以放上連結
   Build these "models" ---> 是指什麼模型？
   "Use Python ..." ---> 太過空洞的描述
   請寫你用什麼工具或技巧或模型，做了什麼事情，達成什麼好的結果
   你寫使用 python 套件與機器學習來訓練與預測模型，這人人都可以寫，完全沒有鑑別度
   頂多只能知道你會最基本的東西
   "Conceptualize, ..." 這句跟 "Use Python ..." 感覺一樣，我沒有看到什麼值得參考的資訊在裡面
   使用 Java 開發，那你的技巧內怎麼沒寫 Java?
   還有不論經驗或是專案，基本上都是你之前做過的東西，請用過去式描述
6. 活動與證照跟所應聘的工作無關

履歷太過空洞，請充實內容，以目前的履歷來說很難拿到電話面試的機會
既然你念了 CS 碩士，總是有些課程專案作業之類的，沒有其他相關經驗就請把課程專案作業寫上去
記得要寫做了什麼？怎麼做的？達成什麼好的結果
