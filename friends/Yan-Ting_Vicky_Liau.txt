我的個人建議
1. 請愛用 PDF
2. 字體大一點，內容太多了，可以斟酌放重要的就好
3. 跟應徵職位沒直接相關的可以省略不放
4. core competency 中每個項目的內容請按照熟悉順序排列，或是重要順序排料
   重要順序不是指妳覺得重要，是公司覺得重要，讓主管一眼就看到
4. 學歷放最後面，列出學位學校地點時間還有 GPA 就好
   我個人覺得 GPA 不重要，但是我們主管會看
   把本來學歷裡面做的東西搬到 working experience 裡面
   空間不夠的話，首先刪學歷，大學時代太古早了可以刪掉
5. kaggle project 沒內容就不要放
6. LinkedIn, Github 請提供超連結可以直接點
7. Ph.D. candidate 下面兩行描述如果履歷的頁面空間不夠，其實可以不用放
   因為主管都只是看過去而已
8. 用 Critiqued 我感覺好像不太合適，我覺得屬於負面字眼。
   用發現，指出等等不帶正負面，或是用正面字眼的字來描述會比較好
